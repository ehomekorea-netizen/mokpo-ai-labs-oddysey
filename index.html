AI Voice Agent 핵심 기술 이해하기

실시간 음성 통신과 도구 호출(Function Calling)이 일어나는 브라우저 내부의 핵심 구조를 시각적으로 학습합니다.

Core Concept

음성 에이전트는 단순한 '텍스트 로봇'이 아닙니다.

실시간 음성 처리(Grok Voice)는 엄청난 양의 오디오 조각들이 끊임없이 왕복하는 고속 파이프라인입니다. 브라우저 마이크를 어떻게 다루고, AI가 스스로 어떻게 도구 주문서(JSON)를 만들며, 쪼개진 오디오를 어떻게 이어붙여서 재생하는지 그 3대 핵심 작동 메커니즘을 아래의 플레이그라운드에서 직접 조작해 보세요.

1. 마이크 오디오 실시간 스트리밍 & WebSocket

실시간 음성 에이전트는 일반적인 HTTP 요청(편지 교환) 대신 **WebSocket(전화 연결)**을 통해 음성 바이너리 데이터를 양방향으로 실시간 스트리밍합니다. 브라우저 마이크의 48,000Hz 오디오 데이터를 xAI 규격인 24,000Hz PCM16 데이터로 다운샘플링하고 Base64로 압축하여 1초에 수십 번씩 보냅니다.

xAI Grok Voice WebSocket 실시간 연결

전화 통화처럼 한 번 연결하여 양방향 오디오를 왕복시키기 위한 웹소켓 레이어를 구현합니다.

왜 HTTP 대신 WebSocket인가요?

질문과 답변이 한 번에 끝나는 일반 API와 다르게, 음성 대화는 0.04초(40ms)마다 오디오 조각이 오고 가야 하므로 **서버 연결을 항상 끊지 않고 유지하는 WebSocket 프로토콜**이 필수적입니다.

1. 주소 정의: wss://api.x.ai/v1/realtime 호출
2. 토큰 주입: 서브프로토콜 헤더에 xai-client-secret.[임시토큰] 탑재
3. 세션 업데이트: 연결 즉시 AI에게 인격 지침 전송
const wsUrl = `wss://api.x.ai/v1/realtime?model=grok-voice-latest`;

ws = new WebSocket(wsUrl, [`xai-client-secret.\${clientToken}`]);



ws.onopen = async () => {

  console.log("WebSocket Connected!");

  await sendSessionUpdate(); // AI 인격 주입

};



ws.onmessage = (e) => {

  handleRealtimeEvent(JSON.parse(e.data));

};

마이크 캡처 및 다운샘플링

마이크로 입력된 48kHz 스테레오 오디오 데이터를 xAI 전용 24kHz 모노 PCM16 데이터로 경량화 변환합니다.

PCM16 변환과 다운샘플링의 당위성

브라우저 기본 마이크 녹음은 오디오 압축률이 없고 48,000Hz로 높아 네트워크 패킷 전송 시 통신 딜레이를 유발합니다. 따라서 데이터 밀도를 절반인 **24,000Hz**로 낮추고 Float32 원시 오디오 배열을 16비트 정수(Int16) 바이너리로 압축해야 실시간 무지연 응답이 보장됩니다.

1. e.inputBuffer.getChannelData(0) 호출
2. 48kHz ➔ 24kHz 프레임 매핑 연산 실행
3. Int16Array 픽셀 정수 매핑 (-32768 ~ 32767)
4. Base64 텍스트 스트링 압축 후 WebSocket 발송
const float32 = e.inputBuffer.getChannelData(0);

const ratio = nativeSampleRate / 24000;

const pcm16 = new Int16Array(downsampledLength);



for (let i = 0; i < downsampledLength; i++) {

  const s = float32[Math.floor(i * ratio)];

  pcm16[i] = s < 0 ? s * 0x8000 : s * 0x7FFF; // Float32 -> Int16 변환

}



const base64 = base64ArrayBuffer(pcm16.buffer);

ws.send(JSON.stringify({

  type: "input_audio_buffer.append",

  audio: base64

}));

오디오 재생 큐 스케줄러

AI가 실시간으로 보낸 잘게 쪼개진 오디오 조각들을 절대 시간 스케줄러로 묶어 끊김 없이 들려줍니다.

왜 예약 스케줄링(Scheduler)을 해야 하나요?

인터넷 지연(Network Jitter) 때문에 오디오 조각들이 도착하는 간격은 항상 다릅니다. 단순히 수신될 때마다 즉시 플레이백하면 버퍼 조각 사이에 0.001초의 공백만 생겨도 뚝뚝 끊기는 노이즈가 발생합니다. 이를 방지하고자 **이전 오디오 조각이 완전히 끝나는 절대 시간(`nextPlaybackTime`)을 연산해 다음 소리 예약 시작 시점을 지정**해야 합니다. 

function drainPlaybackQueue() {

  const buffer = playbackQueue.shift();

  const source = playbackCtx.createBufferSource();

  source.buffer = buffer;



  // 예약 시간이 이미 지난 상태라면 20ms 버퍼 여유를 두고 즉시 스케줄링 시작

  if (nextPlaybackTime <= playbackCtx.currentTime) {

    nextPlaybackTime = playbackCtx.currentTime + 0.02;

  }



  source.start(nextPlaybackTime);

  nextPlaybackTime += buffer.duration; // 다음 조각 재생 예약 위치 업데이트

}

Grok Voice Realtime 에이전트 시스템 아키텍처

클라이언트-서버-인공지능을 잇는 실시간 파이프라인

유튜브 파인더는 브라우저의 오디오 캡처망, Vercel의 프록시 게이트웨이, 그리고 xAI의 실시간 음성 두뇌망이 긴밀하게 얽혀 있는 에이전트입니다.

01
마이크 오디오 프레임 변환

사용자의 목소리를 브라우저가 직접 마이크 스트림으로 캡처한 후, Float32 배열을 48kHz에서 24kHz로 낮추고 PCM16 16비트 인코딩하여 웹소켓 버퍼에 차례로 밀어 넣습니다.

02
WebSocket 양방향 터널

한 번 열리면 통신이 계속 유지되는 웹소켓(WebSocket) 규격 위에 임시 보안 토큰을 태워 xAI와 가상 연결선을 엽니다. 지연 시간이 약 20ms 수준으로 최소화됩니다.

03
Function Calling Gateway

AI가 음성을 듣고 유튜브 검색 주문(JSON)을 쓰면, 브라우저가 이를 납치하여 백엔드 Vercel 게이트웨이를 경유해 YouTube API 결과 카드를 렌더링하고 다시 음성 웰컴 응답을 재생시킵니다.

⚡

WebSocket Realtime vs HTTP REST API 성능 비교

음성 대화에는 왜 일반 API가 부적합할까요?

일반적인 웹 API 통신과 실시간 스트리밍 소켓의 근본적인 차이를 규격 수준에서 이해합니다.

HTTP REST API (일반 웹 요청)

  • 단방향 전송: 클라이언트가 편지를 보내야만 서버가 답장을 쓸 수 있음.
  • 세션 생성 오버헤드: 매 요청 시마다 Handshake를 다시 맺어 레이턴시 대폭 상승.
  • 오디오 전송 불가: 실시간 스트리밍 재생이 불가능하며 파일 전체 다운로드를 기다려야 함.

WebSocket Realtime (양방향 유지)

  • 지속적 상호 교환: 클라이언트와 서버가 항시 연결되어 있어 무지연 양방향 메시지 송신 가능.
  • 20ms 응답성: 지연 시간이 거의 체감되지 않는 음성 통화 환경 완벽 구현.
  • 디코딩 디바이드: 잘게 쪼갠 오디오 청크를 받는 즉시 재생 큐에 스케줄링하여 무지연 플레이백.

instructions.txt와 config.json의 역할

instructions.txt (AI의 인격)

AI 에이전트의 말투, 대답 문장 길이 제약, 특수 문자 필터링, 그리고 특정 사용자의 음성 지시가 왔을 때 어떤 도구(Tool)를 우선 사용할지 서술한 프롬프트 지침입니다.

config.json (하드웨어 제어)

마이크 음성의 감도(VAD Threshold), 입출력 오디오 비트레이트 값, xAI 서버가 수신할 오디오 퀄리티 등 브라우저와 AI 간의 장치 연결 물리 옵션을 정의하는 설정 파일입니다.
